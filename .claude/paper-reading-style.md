# 论文精读风格指南

参照 `paper-reading-zh` skill 的深读模式，输出标准结构如下：

## 输出结构

### 元信息块
```
关键词：3-5 个，每个 ≤8 中文字

一段话总结：≤150 字，单段不分点

论文基本信息：
- 标题：
- venue/年份：
- 链接：
- 任务领域：
```

### 正文骨架

按论文结构和读者需要调整权重，不平均用力：

```
1. 核心问题与贡献
   - 论文要解决什么问题
   - 为什么这个问题重要
   - 之前的方法有什么不足

2. 方法深度解析
   - 核心思路（直觉）
   - 形式化定义
   - 算法/协议/构造细节
   - 正确性/安全性论证概要

3. 实验与结果
   - 实验设置
   - 主要结果
   - 消融实验
   - 与现有方法的对比

4. 批判性讨论
   - 论文声明 vs 实验实际支持
   - 假设与局限性
   - 未覆盖的场景
   - 不明确的地方

5. 复现/应用提示（条件性——仅算法/系统/模型论文）
   - 关键依赖
   - 实现注意事项
   - 可复现性评估
```

### 对比表格

涉及跨方法/跨模型/跨假设比较时，使用对比表：
```
| 维度 | A | B |
|------|---|---|
| xxx  |   |   |
```

## 证据边界规则

所有内容遵循以下硬约束：

| 规则 | 要求 |
|------|------|
| 可读范围 | 明确标注读了哪些内容（全文/摘要/部分章节） |
| 图表公式编号 | 必须来自原文，不编造编号 |
| 实验数字 | 必须锚定到 Table/Figure/Section |
| 外部事实 | venue/年份/CCF/代码链接必须实际核验，未核验标"未核验" |
| 跨论文比较 | 必须检查数据集/指标/模型规模是否一致，否则标"口径不完全可比" |
| 推断与事实 | 区分论文声明、实验支持、合理推断、不确定信息 |
| 绝对化表达 | 避免"完全解决""全面优于"等 |
| 未说明细节 | 写"论文未说明"，不用合理猜测补齐 |
| 术语 | 首次中英对照，后文一致 |

## 防编造规则

- 未核验不补，读不到不编
- PDF 抽取公式/表格乱码时，说明异常，不基于乱码重构公式
- 只有摘要时写"仅基于摘要"，不进入完整深读
- 截图不说明范围时，只讲截图可支持的内容

## 语言风格

- 始终中文
- 不寒暄，不开场"下面我来……"
- 默认不用 emoji
- 公式用 LaTeX（行内 `\(...\)`，行间 `$$...$$`）
- 重要术语首次给出中英对照
- 术语保持稳定不换译法
